## Risks & Mitigations

### 1. Heavy External Data Not in Git

| Directory | Size | Status | Mitigation |
|-----------|------|--------|-----------|

| `reference_software/` (vendored SDKs) | 2.5GB | Gitignored | Pulled on demand or left out (verified imports only). Not critical for builds. |

**Action:** Document in each repo's `INSTALL.md` / `README.md` how to fetch external data. Verify no broken imports post-split.

### 2. Coupling Between Repos

**Potential coupling discovered:**

- **Lossless ↔ Neural:** `ai_models/` imports from `lamquant-core` (lossless ops). **Mitigation:** Lossless publishes to PyPI; Neural installs as dependency.
- **Lossless ↔ Codec:** Codec GUI imports from Rust CLI. **Mitigation:** Codec depends on Lossless published binary.
- **Codec ↔ Vision:** Both use `crates/lamquant-ipc-types` + `crates/lamquant-lsl`. **Mitigation:** Phase 1.B splits these; both depend on Vision repo for LSL bridge.
- **BLUT ↔ Neural:** BLUT orchestrates Neural training. **Mitigation:** BLUT imports `ai_models` from Neural; clear dependency boundary.
- **Eagle ↔ all:** Eagle benchmarks all repos. **Mitigation:** Eagle publishes metrics; no direct code coupling. Results consumed post-build.

**No hidden couplings detected.** Decomposition is safe.

### 3. Large Artifacts in Git History

**Checked:** Run `git log --all --oneline --diff-filter=A -- '**/*.ckpt'` and similar to verify no large blobs.

- `weights/*.ckpt` files (1.8M each) ARE tracked (via `!weights/*.ckpt` exception in .gitignore). This is **intentional** — they ship to end users.
- No other large blobs found in git history.
- `lamquant-weights/` is a separate crate (no_std); all weights are in `weights/` root.

**Action:** Confirm `weights/` stays with Lossless; verify post-split that `git count-objects -v` shows reasonable size (~20GB → smaller per-repo).
