# BIDS mapping profile `bids.1.11.1.single-edf-eeg-events`

- Extends `bids.1.11.1.single-edf-eeg` without redefining that frozen profile.
- Requires a pinned BIDS 1.11.1 `dataset_description.json` and exactly one
  EDF/BDF EEG recording in a portable, duplicate-free dataset tree.
- Projects decoded signal channels, order, rate, labels, and integer samples;
  EDF calibration and absolute recording timing remain capsule-only.
- When the recording has one matching `_events.tsv`, promotes each bounded
  `onset`, `duration`, and `trial_type` field set into an ABIR `Event` on an
  exact recording-relative-seconds clock. Decimal and exponent spellings are
  parsed as exact rationals rather than binary floats.
- An absent events sidecar is valid. Multiple or non-matching event sidecars,
  malformed TSV, negative duration, numeric overflow, and `trial_type` values
  that are `n/a` or cannot be represented as canonical BIDS concept tokens fail
  closed in this bounded profile.
- Preserves every dataset object as a source capsule. Complete event rows,
  additional event columns, and original lexical form remain quarantined and
  recoverable; only the named three-field core receives an exact mapping claim.
- Exact export is available only while the current ABIR interchange identity
  matches the identity bound at import; stale capsules cannot authorize a false
  semantic-equivalence claim.
- The positive fixture is generated by
  `tools/generate_standard_adapter_fixtures.py` and passes BIDS Validator 3.0.1
  with zero errors. This profile remains unpromoted until committed malformed
  event trees and independent expected-rejection receipts are bound.
- Multi-recording datasets, non-EDF/BDF recordings, and other BIDS modalities
  fail closed under this bounded profile.
